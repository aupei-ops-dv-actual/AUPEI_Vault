## How to use this index

- **Read top-down.** Each section is a dependency layer.
- **Treat `Support/` as the bone-yard.** Canon lives in the leaf artifacts + HoR packaging.
- **Every Trailhead (STUB) must specify:** Goal → Must Include → Outputs → Falsifiers → HoR packaging.
- **Reviewer Gates:** Where noted (e.g., “Woit Gate”), those items are *promotion blockers* until satisfied.

## IX. Nomos Logica
**Governance + audit constitution for the Proofield and HoR.**

Nomos Logica (NL) is **derived from the same operator logic** (constraint ζ / extension ω / chance ξ),
but it is **procedural, not axiomatic**.

### Scope
NL exists to enforce and coordinate:
- **Custody + integrity:** HoR manifests, hashing standards, provenance, packaging rules.
- **Replication discipline:** dispute handling, replication conflicts, audit escalation paths.
- **Promotion gates:** STUB → Appx → AoC → Canon, including downgrade controls.
- **Stewardship:** roles/councils/immune-system structure (AUPEI-aligned), licensing/rights constraints, and staged-consent interfaces for non-human agents (when applicable).

### Non-scope
NL must **never** be used to:
- Introduce or modify physics assumptions, domain declarations, or operator definitions.
- Relax epistemic requirements (Units Firewall, discriminator enforcement, falsifiers, reproducibility).

### Jurisdiction Map
- **AoC/PF:** defines what is true (physics, operators, constants)
- **Nomos Logica:** defines how truth is recorded, audited, promoted
- **AUPEI Protocols:** defines how agents interact (initialization, memory, yield)

### Conflict resolution
- If AoC/PF says **STOP** (yield, falsified, underdetermined) → NL cannot override.
- If NL says **inadequate HoR packaging** → AoC/PF must comply (physics content unchanged).

- `Appx_26_On_Nomos_Logica_PF_vX.X` — the Charter
  - Must include (procedural only):
    - Promotion rules (PF/AoC gates) as references to `AoC_00` / PF requirements *(no restating physics law)*
    - HoR custody + hashing + manifest standards + replication dispute protocol
    - Enforcement roles / councils / immunity model (AUPEI-aligned)
    - Rights + staged-consent interfaces for non-human agents (if applicable)
  - `Appx_26_NomosLogica_Support/` *(legacy drafts, examples, engine tests, council notes)*

**Yield Supremacy Rule:** NL does not create exceptions to AoC/PF falsifiers, discriminator failures, Units Firewall violations, or missing dependencies.  
NL governs **recording, adjudication, custody, and promotion gates only**.

**Collapse consequence**
> If governance and audit law fail, truth capture occurs: replication becomes political, promotion becomes corruptible, and the HoR loses meaning.
